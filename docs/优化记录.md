# LSMTree ScanColumn 优化

查询语句：`SELECT sum(score) FROM test;`（1000 万行数据）

## 性能数据

| 优化阶段 | 查询时间 | 提升倍数 |
|---------|---------|---------|
| 初始 (SkipList MemTable) | ~0.6s | - |
| 批量 AddRows 替代逐行 AddRow | ~0.17s | 3.5x |
| 键值分离 + 整数主键特化 | **0.079s** | **7.6x** |

最终吞吐量：**1000 万行 / 0.079s ≈ 1.27 亿行/秒**

## 优化 1：VectorizedMemTable 替换 SkipList

### 问题
SkipList 每次插入 `new` 一个节点，`malloc` 开销大，指针跳转缓存不友好。

### 方案
- 使用 `Arena` 连续内存分配器（bump allocation），数据紧凑存储
- 用 `vector<uint32_t> offsets_` 记录每条记录的偏移量
- 写入 O(1) 追加，读取时延迟排序（`EnsureSorted`）

### 数据布局
```
Arena: [key_len: 4B][value_len: 4B][key_data][value_data] ...
offsets_: [offset0, offset1, offset2, ...]
```

## 优化 2：零拷贝列扫描

### 问题
`ScanColumn` 遍历 MemTable/Immutable 时，通过 `Slice{std::string(...)}` 产生大量内存拷贝。

### 方案
- 新增 `RowCodec::DecodeColumnRaw()`：直接返回 `(const Byte*, uint32_t)` 指针对，不分配内存
- 新增 `VectorizedMemTable::GetValueRawByIndex()`：通过索引直接访问 Arena 中的 value 数据
- 全程使用裸指针传递，零拷贝解析列值

## 优化 3：批量 AddRows 替代逐行 AddRow

### 问题
perf 火焰图显示 `AddRow` 占 BuildSelectionVector 的 **23.5%**，逐行函数调用开销大。

### 方案
- 按 `(source, source_id)` 分组收集行索引到 `vector<uint32_t>`
- 遍历完成后一次性调用 `sv.AddRows()` 批量添加
- SSTable 整块无冲突时用 `sv.AddContiguous()` 更高效

## 优化 4：键值分离 + 整数主键特化

### 问题
排序时 key 和 value 混合存储在同一块 Arena 中，排序需要通过 offset 间接访问 key，且会污染存放 value 的缓存行。

### 方案

**键值分离**：
```
之前: Arena [key_len][value_len][key_data][value_data]
之后: key_arena_:   [key0][key1]...       (仅字符串主键)
      value_arena_: [value0][value1]...   (排序时完全不访问)
```

**整数主键特化**：
```cpp
struct IntEntry {       // 16B，缓存行可容纳 4 个
  int key;              // 直接存 key 值
  uint32_t value_offset;
  uint32_t value_len;
  uint32_t seq;         // 插入序号，保证稳定排序
};
```

- 整数主键排序时直接比较 `int`，**0 内存间接访问**
- SSTable 去重时也直接解引用 `int*`，无需 `GetColumnValuePointer` 通用路径
- 按主键类型分派：`BuildSelectionVectorInt()` / `BuildSelectionVectorDouble()` / `BuildSelectionVectorString()`

## 优化 5：微观优化

- `OffsetCompare` 中 key 相同时比较 `offset`/`seq`，使 `std::sort` 等价于稳定排序
- `GetKeyAt`/`GetValueAt` 使用 `*reinterpret_cast<const uint32_t*>(ptr)` 直接解引用替代 `memcpy`
- `Serilize()` 预分配 `res.reserve(approximate_size_)` 避免 string 多次扩容
- `ColumnVector::Reserve(total_rows)` 预分配列容器容量减少 realloc
- ZoneMap 剪枝：SSTable RowGroup 范围与 mem_keys 不重叠时直接跳过去重

## 核心思想

```
SkipList (指针跳转)  →  Arena + vector (连续内存)
通用 key 比较        →  int 直接比较 (主键类型特化)
key+value 混合存储   →  键值分离 (排序时不碰 value)
逐行 AddRow          →  批量 AddRows
string 拷贝          →  裸指针零拷贝
```
